HLD, LLD & Implementation Blueprint

Khảo sát sâu lý thuyết nền tảng, thiết kế mức cao (HLD), thiết kế chi tiết (LLD), cấu trúc lớp và cách lập trình chuyển đổi hệ thống sang Spring Boot.

Đặc tả thiết kế:
Tài liệu này bao gồm Lý thuyết thiết kế nền tảng, Thiết kế mức cao (HLD), Thiết kế mức chi tiết (LLD) và Mã nguồn Java đầy đủ để triển khai hệ thống đồng bộ dữ liệu MongoDB sang Elasticsearch.
Lý Thuyết Nền Tảng

1. Cơ sở Lý thuyết Thiết kế Hệ thống

A. Keyset Pagination vs Offset Pagination

Trong phân trang kiểu Offset truyền thống (LIMIT M OFFSET N), để lấy dữ liệu trang thứ $K$, database bắt buộc phải duyệt qua toàn bộ $N$ bản ghi trước đó, sắp xếp và vứt chúng đi, chỉ trả về $M$ bản ghi cuối cùng. Độ phức tạp thuật toán là $O(N)$. Khi dữ liệu đạt hàng triệu bản ghi, I/O của database sẽ bị quá tải nghiêm trọng.

Giải pháp Keyset Pagination (Cursor-based): Thay vì bỏ qua $N$ bản ghi, chúng ta sử dụng cột khóa chính có tính chất tuần tự (như _id ObjectId trong MongoDB) làm điểm neo (Cursor). Câu truy vấn trở thành WHERE _id > last_id ORDER BY _id ASC LIMIT M. Cơ sở dữ liệu sẽ nhảy trực tiếp tới điểm bắt đầu bằng chỉ mục B-Tree trong thời gian $O(\log N)$, loại bỏ hoàn toàn việc scan đĩa cứng lãng phí.

B. Change Data Capture (CDC) vs Database Polling

Để đồng bộ thời gian thực, nếu dùng phương pháp Polling (quét định kỳ bảng dữ liệu dựa theo cột updated_at), hệ thống sẽ gây nghẽn lock và tiêu tốn CPU của database ngay cả khi không có dữ liệu thay đổi.

Giải pháp CDC (lắng nghe Oplog): MongoDB ghi lại tất cả các hoạt động thay đổi dữ liệu vào một tệp nhật ký giao dịch tuần tự gọi là Oplog (Operations Log). Change Stream cho phép ứng dụng đăng ký lắng nghe tệp tin này dưới dạng luồng sự kiện (Event-driven CDC). Cơ chế này giúp tối ưu tài nguyên database chính gần như về mức $O(1)$ đối với các giao dịch đọc, và chỉ trigger xử lý khi có sự kiện ghi thực tế phát sinh.

C. Project Reactor & Reactive Buffering

Trong Java Spring Boot, việc ghi lẻ tẻ từng bản ghi Change Stream lên Elasticsearch sẽ phá hủy hiệu năng ghi (Ingestion rate) của công cụ tìm kiếm. Chúng ta sử dụng luồng phản ứng (Reactive Streams) của Project Reactor (Spring WebFlux).

Toán tử bufferTimeout(batchSize, timeout) cung cấp một bộ đệm thread-safe trong bộ nhớ JVM. Nó gom các phần tử từ Change Stream và flush hàng loạt dưới dạng Bulk Operation khi:

  • Số lượng sự kiện trong hàng đợi đạt ngưỡng tối đa (ví dụ 1000 docs).
  • Hoặc khoảng thời gian chờ tối đa trôi qua kể từ phần tử đầu tiên (ví dụ 2 giây).

3. Thiết Kế Mức Chi Tiết (LLD)

A. Đặc tả cấu trúc Redis Checkpoints (Redis Keys)

Hệ thống quản lý trạng thái đồng bộ thông qua các key trong Redis dưới dạng chuỗi (String):

Key Name (Redis) Data Type Value Format Mô tả chức năng
checkpoint:users String MongoDB ObjectId (Ví dụ: 64f21b7e9f3b1021bc8201a2) hoặc Oplog clusterTime (Unix Timestamp - Long) Lưu điểm neo cuối cùng của tiến trình đồng bộ Users.
checkpoint:messages String MongoDB ObjectId hoặc Oplog clusterTime (Unix Timestamp - Long) Lưu điểm neo cuối cùng của tiến trình đồng bộ Messages.

B. Cấu hình đa luồng ghi song song (ThreadPool Configurations)

Để pha Initial Load (Backfile) đạt hiệu năng tối đa mà không gây tràn bộ nhớ (Out Of Memory - OOM), cấu hình ThreadPoolExecutor sử dụng chiến lược áp lực ngược như sau:

  • corePoolSize: 4 (Số lượng luồng xử lý core cố định).
  • maxPoolSize: 8 (Số lượng luồng tối đa khi hàng đợi quá tải).
  • queueCapacity: 50 (Giới hạn số lượng lô dữ liệu nằm chờ trong RAM).
  • RejectedExecutionHandler: CallerRunsPolicy.
    Tại sao dùng CallerRunsPolicy? Khi Elasticsearch ghi chậm dẫn đến hàng đợi ThreadPool (capacity=50) bị đầy, chính thread chính (luồng đọc dữ liệu từ MongoDB) sẽ đứng ra thực thi tác vụ ghi đó. Việc này tự động dừng luồng đọc Mongo lại, tạo cơ chế giảm tốc tự nhiên (Backpressure), ngăn chặn OOM trên JVM.

C. Ánh xạ Dữ liệu Chi tiết (Mongo BSON to ES Document Mapping)

Đặc tả các quy tắc chuyển dịch trường thông tin (Field Transformation Rules):

MongoDB Field (Source) Elasticsearch Field (Target) Transform Rule / Algorithm
_id (ObjectId) _id (String) Chuyển thành dạng Hex String (Đảm bảo tính Idempotency).
username (String) username (String) Chuyển thành chữ thường: doc.getString("username").toLowerCase().
tags (Array) active (Integer) Mặc định = 1 nếu có username. Nếu tags chứa banned hoặc hidden → active = 0.
tags (Array) creator (Integer) Nếu tags chứa signed hoặc stats → creator = 1, ngược lại = 0.
tags (Array) badge_count (Integer) Đếm số lượng tag trùng khớp với Regex: auto-follow|badge|event|famous|featured|lang|signed|stats.
tags (Array) gender (Array) Lọc các phần tử mảng bắt đầu bằng tiền tố gender:.

Dùng Redis để lưu trữ offset dạng ObjectId cho Backfile hoặc clusterTime (Timestamp) cho Change Stream:

StateService.java 
@Service
public class StateService {
    @Autowired
    private StringRedisTemplate redisTemplate;

    public String getState(String key) {
        return redisTemplate.opsForValue().get(key);
    }

    public void setState(String key, String value) {
        redisTemplate.opsForValue().set(key, value);
    }

    public void clearState(String key) {
        redisTemplate.delete(key);
    }
}

6. Bộ Khung Đồng Bộ Hóa Trừu Tượng (BaseSyncService.java)

Sử dụng thiết kế lớp trừu tượng kết hợp Spring Data và Reactive Streams (Project Reactor) để tự động hóa quá trình chuyển đổi giữa pha nạp lịch sử (Backfile) và CDC (Change Stream):

BaseSyncService.java Generic Core Processing Logic 
public abstract class BaseSyncService<T> {
    @Autowired
    protected MongoTemplate mongoTemplate;
    @Autowired
    protected ReactiveMongoTemplate reactiveMongoTemplate;
    @Autowired
    protected ElasticsearchClient esClient;
    @Autowired
    protected StateService stateService;

    protected abstract String getIndexName();
    protected abstract String getCollectionName();
    protected abstract String getStateKey();
    protected abstract BulkOperation transformToBulkAction(Document doc);
    protected abstract List<BulkOperation> processChangeStreamEvent(ChangeStreamEvent<Document> event);

    public void runSync() {
        boolean indexCreated = ensureIndex();
        if (indexCreated) {
            stateService.clearState(getStateKey());
        }

        String checkpoint = stateService.getState(getStateKey());
        if (checkpoint == null) {
            log.info("Bắt đầu đồng bộ mới (Fresh Initial Load)...");
            Instant startTime = Instant.now();
            runInitialLoad();
            startChangeStream(startTime);
        } else if (isObjectId(checkpoint)) {
            log.info("Tiếp tục chạy Initial Load từ ID: " + checkpoint);
            Instant startTime = Instant.now();
            resumeInitialLoad(new ObjectId(checkpoint));
            startChangeStream(startTime);
        } else {
            log.info("Tiếp tục chạy Change Stream từ Timestamp: " + checkpoint);
            resumeChangeStream(Long.parseLong(checkpoint));
        }
    }

    // Pha 1: Initial Load (Quét Keyset)
    protected void runInitialLoad() {
        optimizeIndexSettings();
        ObjectId lastId = null;
        int batchSize = 1000;
        
        while (true) {
            Query query = new Query();
            if (lastId != null) {
                query.addCriteria(Criteria.where("_id").gt(lastId));
            }
            query.with(Sort.by(Sort.Direction.ASC, "_id")).limit(batchSize);

            List<Document> docs = mongoTemplate.find(query, Document.class, getCollectionName());
            if (docs.isEmpty()) break;

            executeParallelBulk(docs);
            lastId = docs.get(docs.size() - 1).getObjectId("_id");
            stateService.setState(getStateKey(), lastId.toHexString());
        }
        
        restoreIndexSettings();
    }

    // Pha 2: Lắng nghe CDC với bufferTimeout
    protected void startChangeStream(Instant startAt) {
        ChangeStreamOptions options = ChangeStreamOptions.builder()
            .resumeAt(startAt)
            .build();

        reactiveMongoTemplate.changeStream(getCollectionName(), options, Document.class)
            .flatMapIterable(event -> processChangeStreamEvent(event))
            .bufferTimeout(500, Duration.ofSeconds(2))
            .subscribe(
                actions -> flushBulk(actions),
                error -> handleStreamError(error)
            );
    }

    protected void flushBulk(List<BulkOperation> actions) {
        if (actions.isEmpty()) return;
        try {
            esClient.bulk(b -> b.operations(actions));
        } catch (IOException e) {
            log.error("Lỗi ghi bulk lên Elasticsearch", e);
        }
    }

    private void handleStreamError(Throwable error) {
        if (error instanceof MongoCommandException ex) {
            if (ex.getErrorCode() == 136) {
                log.warn("Oplog rollover detected! Re-triggering Initial Load...");
                stateService.clearState(getStateKey());
                runSync(); // Chạy lại từ đầu
                return;
            }
        }
        log.error("Lỗi Change Stream không xác định: ", error);
    }

    // Đổi cấu hình ghi tốc độ cao
    private void optimizeIndexSettings() { ... }
    private void restoreIndexSettings() { ... }
    private boolean ensureIndex() { ... }
}
Bước 4: User Sync

7. Triển Khai Cụ Thể Cho User (UserSyncService.java)

Kế thừa BaseSyncService, thực hiện trích xuất dữ liệu, định cấu hình cờ creator, tính toán badge_count và xử lý tag giới tính:

UserSyncService.java 
@Service
public class UserSyncService extends BaseSyncService<Document> {

    @Override
    protected String getIndexName() { return "user-index"; }

    @Override
    protected String getCollectionName() { return "user"; }

    @Override
    protected String getStateKey() { return "checkpoint:users"; }

    @Override
    protected BulkOperation transformToBulkAction(Document doc) {
        String userId = doc.getObjectId("_id").toHexString();
        String username = doc.getString("username");
        String displayName = doc.getString("display_name");
        List<String> tags = doc.getList("tags", String.class, Collections.emptyList());

        Map<String, Object> source = new HashMap<>();
        source.put("user_id", userId);
        source.put("username", username != null ? username.toLowerCase() : null);
        source.put("display_name", displayName);

        int active = username != null ? 1 : 0;
        int creator = 0;
        int badgeCount = 0;
        List<String> gender = new ArrayList<>();

        for (String tag : tags) {
            if (tag.startsWith("gender:")) {
                gender.add(tag);
            }
            if (tag.matches("auto-follow|badge|event|famous|featured|lang|signed|stats")) {
                badgeCount++;
            }
            if (tag.equals("banned") || tag.equals("hidden")) {
                active = 0;
            }
            if (tag.equals("signed") || tag.equals("stats")) {
                creator = 1;
            }
        }

        source.put("active", active);
        source.put("creator", creator);
        source.put("badge_count", badgeCount);
        source.put("gender", gender.isEmpty() ? null : gender);

        return new BulkOperation.Builder()
            .index(i -> i.index(getIndexName()).id(userId).document(source))
            .build();
    }

    @Override
    protected List<BulkOperation> processChangeStreamEvent(ChangeStreamEvent<Document> event) {
        String operation = event.getOperationType().getValue();
        String docId = event.getDocumentKey().getObjectId("_id").toHexString();

        if (operation.equals("delete")) {
            return List.of(new BulkOperation.Builder()
                .delete(d -> d.index(getIndexName()).id(docId))
                .build());
        }

        Document fullDoc = event.getBody();
        if (fullDoc != null) {
            return List.of(transformToBulkAction(fullDoc));
        }
        return Collections.emptyList();
    }
}
Bước 5: Khởi Chạy

8. Trình Khởi Chạy CLI (SyncApplication.java)

Sử dụng CommandLineRunner để cấu hình tham số truyền vào giúp bạn dễ dàng chạy file JAR phân tách tiến trình cho users hoặc messages:

SyncApplication.java 
@SpringBootApplication
public class SyncApplication implements CommandLineRunner {
    @Autowired
    private UserSyncService userSyncService;

    public static void main(String[] args) {
        SpringApplication.run(SyncApplication.class, args);
    }

    @Override
    public void run(String... args) throws Exception {
        if (args.length > 0 && args[0].equals("users")) {
            userSyncService.runSync();
        } else {
            System.out.println("Vui lòng chạy với đối số: 'users' hoặc 'messages'");
        }
    }
}
