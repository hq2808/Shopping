Hướng dẫn sử dụng Postman Collection

Tài liệu này hướng dẫn QA / Tester cách sử dụng bộ Postman Collection có sẵn để kiểm thử toàn bộ các REST API của hệ thống Mini Shopee. Collection bao gồm đầy đủ các nhóm endpoint từ xác thực, sản phẩm, đặt hàng đến quản lý người dùng — tất cả đã được cấu hình sẵn Bearer Token và Environment Variables để giúp bạn bắt đầu kiểm thử ngay lập tức.

✅ Không cần code 🔐 JWT Auth có sẵn ⚡ Cả Happy & Negative path

📥 Bước 1: Import Collection vào Postman

Thực hiện theo các bước dưới đây để nhập Collection và thiết lập môi trường làm việc trong Postman.

🗂️ Tải file Collection

Bộ Postman Collection được đính kèm trực tiếp trong cổng tài liệu này. Nhấp vào nút dưới đây để tải về máy của bạn:

📥 Tải Postman Collection JSON
📁 Đường dẫn trong Git repo: docs/qa/mini_shopee_postman_collection.json

🚀 Các bước Import vào Postman

  1. Mở ứng dụng Postman trên máy tính.
  2. Nhấn nút "Import" ở góc trên bên trái (cạnh logo Postman).
  3. Trong hộp thoại Import, kéo và thả file mini_shopee_postman_collection.json vào vùng "Drop files here", hoặc nhấn "Browse Files" để chọn file từ máy tính.
  4. Nhấn "Import" để xác nhận.
  5. Sau khi import thành công, bộ collection 🛒 Mini Shopee API sẽ xuất hiện ở thanh bên trái (mục Collections).

⚙️ Thiết lập Environment Variables

Để collection hoạt động đúng, bạn cần tạo một Environment trong Postman với các biến sau:

  1. Nhấn vào biểu tượng 🌍 Environments ở thanh bên trái.
  2. Nhấn "Create Environment" → Đặt tên là Mini Shopee Local.
  3. Thêm các biến theo bảng bên dưới.
  4. Nhấn "Save" rồi chọn environment này ở góc phải trên của Postman.
Tên biến Kiểu Giá trị ban đầu Mô tả
base_url String http://localhost:8080 URL gốc của server Mini Shopee đang chạy local
jwt_token String (để trống) JWT Bearer Token sau khi đăng nhập thành công, dùng cho tất cả API có Auth
💡 Lưu ý: Tất cả request trong Collection đã sử dụng {{base_url}} và {{jwt_token}} — bạn chỉ cần đặt đúng giá trị trong Environment là xong.

🔐 Bước 2: Xác thực & Lấy JWT Token

Hầu hết các API của Mini Shopee yêu cầu xác thực bằng JWT Bearer Token. Bạn cần đăng nhập trước để lấy token, sau đó gán vào biến môi trường.

1️⃣ Gọi API Đăng nhập

Thực hiện request sau để lấy JWT token:

Method POST
Endpoint {{base_url}}/api/auth/login
Content-Type application/json
Auth ❌ Không cần

Request Body:

{
  "username": "user",
  "password": "password"
}

Response mẫu (200 OK):

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJ1c2VyIiwia...",
  "type": "Bearer",
  "expiresIn": 86400
}

2️⃣ Gán Token vào Environment

  1. Sau khi nhận response, sao chép giá trị của trường token.
  2. Mở Environments → Chọn Mini Shopee Local.
  3. Dán token vào cột Current Value của biến jwt_token.
  4. Nhấn "Save".
  5. Từ đây, tất cả request yêu cầu Auth sẽ tự động gửi header Authorization: Bearer {{jwt_token}}.
✅ Collection đã cấu hình sẵn Bearer Token tại tab Authorization của từng request (loại: Bearer Token, giá trị: {{jwt_token}}).

👥 Tài khoản mặc định để kiểm thử

Username Password Role Quyền truy cập
user password ROLE_USER Xem sản phẩm, đặt hàng, xem profile & lịch sử đơn hàng
admin password ROLE_ADMIN Toàn bộ quyền của USER + quản trị hệ thống, quản lý sản phẩm
⚠️ Quan trọng: Đây là tài khoản seed data dành cho môi trường phát triển. Tuyệt đối không sử dụng thông tin này trong môi trường production.

📋 Bước 3: Danh sách API theo nhóm

Toàn bộ các endpoint trong Collection được tổ chức thành 4 nhóm chính. Các nhóm được hiển thị dưới dạng folder trong Postman.

Nhóm Method Endpoint Mô tả Auth cần?
🔑 Auth POST /api/auth/login Đăng nhập lấy JWT token ❌ Không
🔑 Auth POST /api/auth/register Đăng ký tài khoản người dùng mới ❌ Không
🛍️ Products GET /api/products Lấy danh sách tất cả sản phẩm (có phân trang) ✅ Có
🛍️ Products GET /api/products/{id} Lấy chi tiết một sản phẩm theo ID ✅ Có
📦 Orders POST /api/orders Đặt hàng đồng bộ (Pessimistic Lock – an toàn cao) ✅ Có
📦 Orders POST /api/orders/async Đặt hàng bất đồng bộ (Redis + Message Queue – hiệu suất cao) ✅ Có
📦 Orders GET /api/orders Xem lịch sử đơn hàng của người dùng hiện tại ✅ Có
👤 Users GET /api/users/profile Lấy thông tin profile & số dư ví của người dùng hiện tại ✅ Có

✅ Bước 4: Luồng kiểm thử chuẩn (Happy Path)

Thực hiện các bước theo thứ tự dưới đây để hoàn thành một luồng kiểm thử end-to-end cơ bản cho chức năng đặt hàng của Mini Shopee.

1

Đăng nhập & Lấy JWT Token

Gọi POST /api/auth/login với tài khoản user / password.

POST {{base_url}}/api/auth/login
Content-Type: application/json

{
  "username": "user",
  "password": "password"
}

➡️ Sao chép giá trị token từ response → Gán vào biến {{jwt_token}} trong Environment.

2

Lấy danh sách sản phẩm

Gọi GET /api/products để xem toàn bộ sản phẩm có sẵn.

GET {{base_url}}/api/products
Authorization: Bearer {{jwt_token}}

➡️ Ghi nhớ productId của sản phẩm bạn muốn mua (ví dụ: id: 2). Kiểm tra trường stock phải > 0.

3

Đặt hàng

Gọi POST /api/orders với danh sách sản phẩm muốn mua.

POST {{base_url}}/api/orders
Authorization: Bearer {{jwt_token}}
Content-Type: application/json

{
  "items": [
    {
      "productId": 2,
      "quantity": 1
    }
  ]
}
4

Kiểm tra Response đặt hàng

Xác nhận response trả về thành công:

Trường cần kiểm traGiá trị mong đợi
HTTP Status Code200 OK
idSố nguyên dương (orderId mới tạo)
statusCONFIRMED hoặc PENDING
totalAmountTổng tiền đúng với giá sản phẩm × số lượng
5

Xác nhận lịch sử đơn hàng

Gọi GET /api/orders để xác nhận đơn hàng xuất hiện trong lịch sử.

GET {{base_url}}/api/orders
Authorization: Bearer {{jwt_token}}

➡️ Kiểm tra đơn hàng vừa tạo xuất hiện trong mảng kết quả với đúng orderId, status và totalAmount.

6

Kiểm tra số dư ví

Gọi GET /api/users/profile để xác nhận số dư ví đã bị trừ đúng.

GET {{base_url}}/api/users/profile
Authorization: Bearer {{jwt_token}}

➡️ Trường walletBalance phải giảm đúng bằng totalAmount của đơn hàng vừa đặt. Nếu không khớp → Ghi nhận bug về logic thanh toán.

🚫 Bước 5: Kiểm thử trường hợp ngoại lệ (Negative Cases)

Các kịch bản sau đây kiểm tra khả năng xử lý lỗi của hệ thống. Mỗi kịch bản phải trả về đúng HTTP status code và message lỗi rõ ràng.

# Kịch bản Request Expected Response Ghi chú
1 Không có token Gọi GET /api/products không có header Authorization 401 Unauthorized Kiểm tra message: "Unauthorized" hoặc "Full authentication is required"
2 Token sai / hết hạn Header: Bearer token_sai_abc123 401 Unauthorized Hệ thống phải từ chối token không hợp lệ
3 Tồn kho bằng 0 POST /api/orders với sản phẩm đã hết hàng (stock = 0) 400 Bad Request Message: "Insufficient stock" hoặc tương đương
4 Số dư ví không đủ POST /api/orders với tổng giá trị vượt quá số dư trong ví 400 Bad Request Message: "Insufficient balance" hoặc tương đương
5 productId không tồn tại POST /api/orders với "productId": 99999 404 Not Found Message: "Product not found"
💡 Tip: Postman cho phép bạn viết Tests Script (tab Tests) để tự động kiểm tra status code & nội dung response. Ví dụ: 
pm.test("Status is 401", () => {
    pm.response.to.have.status(401);
});
pm.test("Error message exists", () => {
    pm.expect(pm.response.json()).to.have.property("message");
});

💡 Mẹo & Lưu ý

Một số lưu ý quan trọng giúp quá trình kiểm thử diễn ra suôn sẻ và nhất quán.

⏰ Token hết hạn: JWT Token có thời gian sống là 24 giờ (expiresIn: 86400). Nếu gặp lỗi 401 sau khi đã đặt token, hãy thực hiện lại bước đăng nhập để lấy token mới và cập nhật biến jwt_token trong Environment.
🖥️ Server phải chạy trước: Đảm bảo ứng dụng Mini Shopee đang chạy tại http://localhost:8080 trước khi thực hiện bất kỳ request nào. Kiểm tra bằng cách truy cập http://localhost:8080/actuator/health — nếu trả về {"status":"UP"} là server đã sẵn sàng.
🔄 Reset dữ liệu sau mỗi lần test: Sau các bài test đặt hàng, tồn kho và số dư ví có thể thay đổi, ảnh hưởng đến các test case tiếp theo. Sử dụng các lệnh sau để reset về trạng thái ban đầu:

🗄️ Reset tồn kho bằng SQL

Kết nối vào database MySQL/PostgreSQL và chạy lệnh:

-- Reset tồn kho về 100 cho tất cả sản phẩm
UPDATE products SET stock = 100;

-- Hoặc reset cho một sản phẩm cụ thể (ví dụ: id = 1)
UPDATE products SET stock = 100 WHERE id = 1;

-- Kiểm tra sau khi reset
SELECT id, name, stock FROM products;

⚡ Reset tồn kho trong Redis Cache

Nếu hệ thống sử dụng Redis để cache tồn kho (cho luồng đặt hàng bất đồng bộ), bạn cần reset cả dữ liệu trong Redis: 

# Reset tồn kho sản phẩm id=1 về 100
redis-cli SET product:stock:1 100

# Kiểm tra giá trị hiện tại
redis-cli GET product:stock:1

# Reset tất cả (CẢNH BÁO: Xóa toàn bộ dữ liệu Redis!)
redis-cli FLUSHDB
⚠️ Cảnh báo: Lệnh FLUSHDB sẽ xóa toàn bộ dữ liệu trong Redis database hiện tại. Chỉ dùng trong môi trường development.

💰 Reset số dư ví người dùng

-- Reset số dư ví của user thành 1,000,000
UPDATE users SET wallet_balance = 1000000 WHERE username = 'user';

-- Kiểm tra sau khi reset
SELECT username, wallet_balance FROM users;
