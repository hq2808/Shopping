QA / Tester Deployment

Hướng Dẫn Khởi Chạy Hệ Thống
(Quick Start Guide)

Tài liệu hướng dẫn chi tiết dành cho QA/Tester để thiết lập, tải ảnh docker từ xa và kích hoạt toàn bộ cụm ứng dụng kiểm thử trong 2 phút.

Yêu cầu

1. Chuẩn bị hệ thống

Hệ thống đã được đóng gói và tối ưu hóa hoàn toàn dưới dạng container Docker. Tester hoặc bất kỳ thành viên nào trong nhóm không cần phải cài đặt Java JDK, Maven, hoặc cơ sở dữ liệu PostgreSQL trực tiếp lên máy.

Yêu cầu duy nhất:
  • Máy tính của bạn cần cài đặt ứng dụng **Docker Desktop** (Tải về miễn phí tại trang chủ: Docker Desktop).
  • Đảm bảo rằng phần mềm Docker Desktop đã được mở và đang chạy bình thường trên máy tính của bạn trước khi thực hiện các bước sau.
Quy trình

2. Các bước khởi chạy chi tiết

Bước 2.1: Tạo thư mục làm việc

Tạo một thư mục mới hoàn toàn trên máy tính của bạn để chứa tệp tin cấu hình (Ví dụ: C:\MiniShopee hoặc thư mục tương tự).

Bước 2.2: Tạo tệp tin cấu hình docker-compose

Tạo một tệp tin trống mới tên là docker-compose.yml bên trong thư mục vừa tạo. Mở tệp tin bằng Notepad hoặc bất kỳ trình soạn thảo văn bản nào để dán cấu hình.

Bước 2.3: Tạo tệp kích hoạt nhanh tự động (Khuyên dùng - Auto Start)

LỰA CHỌN A: Dành cho Windows (Chạy bằng một cú click chuột)

Tạo thêm một tệp tin mới tên là run.bat nằm ngay bên cạnh tệp docker-compose.yml. Mở tệp bằng Notepad, dán toàn bộ đoạn mã tự động sau vào và lưu lại: 

@echo off
title Mini Shopee - Quick Start Tool
color 0B
echo [1/4] Dang don dep cac container cu de tranh xung dot...
docker stop shopee_mini_db shopee_mini_app shopee_mini_redis shopee_mini_rabbitmq >nul 2>&1
docker rm shopee_mini_db shopee_mini_app shopee_mini_redis shopee_mini_rabbitmq >nul 2>&1
echo [2/4] Dang tu dong khoi chay ung dung...
docker compose up -d
if %ERRORLEVEL% NEQ 0 (
    color 0C
    echo [LOI] Vui long mo phan mem "Docker Desktop" truoc khi chay!
    pause
    exit /b
)
echo [3/4] Dang cho ung dung khoi dong (5 giay)...
timeout /t 5 /nobreak >nul
echo [4/4] Dang mo trinh duyet...
start http://localhost:8080
color 0A
echo KICH HOAT THANH CONG!
pause >nul

Bạn chỉ cần **kích đúp chuột (double-click) vào tệp run.bat** để kích hoạt tự động toàn bộ hệ thống!

LỰA CHỌN B: Dành cho Ubuntu / Linux (Chạy bằng một dòng lệnh Bash)

Tạo thêm một tệp tin mới tên là run.sh nằm ngay bên cạnh tệp docker-compose.yml. Mở tệp run.sh lên bằng trình soạn thảo văn bản và dán toàn bộ mã Bash sau vào rồi lưu lại: 

#!/bin/bash
echo "[1/4] Dang don dep cac container cu de tranh xung dot..."
docker stop shopee_mini_db shopee_mini_app shopee_mini_redis shopee_mini_rabbitmq >/dev/null 2>&1
docker rm shopee_mini_db shopee_mini_app shopee_mini_redis shopee_mini_rabbitmq >/dev/null 2>&1
echo "[2/4] Dang tu dong khoi chay ung dung..."
if command -v docker-compose &> /dev/null; then
    docker-compose up -d
else
    docker compose up -d
fi
if [ $? -ne 0 ]; then
    echo "[LOI] Vui long dam bao rang Docker Service dang hoat dong!"
    exit 1
fi
echo "[3/4] Dang cho ung dung khoi dong (5 giay)..."
sleep 5
echo "[4/4] Dang mo cong tai lieu kiem thu..."
if command -v xdg-open &> /dev/null; then
    xdg-open http://localhost:8080
else
    echo "Mo trinh duyet va truy cap: http://localhost:8080"
fi

Mở Terminal tại thư mục này, thực hiện cấp quyền thực thi và chạy file Bash bằng lệnh duy nhất: 

chmod +x run.sh && ./run.sh

Hệ thống sẽ tự động dọn dẹp các container bị trùng tên (tránh xung đột), khởi chạy Database, kích hoạt Server và tự động mở trình duyệt web dẫn thẳng vào cổng tài liệu sau 5 giây.

Docker Compose File

3. Nội dung cấu hình mẫu (docker-compose.yml)

Dưới đây là tệp tin cấu hình tiêu chuẩn đã thiết lập sẵn môi trường bao gồm PostgreSQL, Redis, RabbitMQ và ứng dụng: 

version: '3.8'
services:
  postgres-db:
    image: postgres:16-alpine
    container_name: shopee_mini_db
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: sa_user
      POSTGRES_PASSWORD: sa_password
      POSTGRES_DB: shopee_mini
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U sa_user -d shopee_mini"]
      interval: 5s
      timeout: 5s
      retries: 5
    networks:
      - shopee-net

  redis:
    image: redis:7-alpine
    container_name: shopee_mini_redis
    ports:
      - "6379:6379"
    volumes:
      - redisdata:/data
    networks:
      - shopee-net

  rabbitmq:
    image: rabbitmq:3.13-management-alpine
    container_name: shopee_mini_rabbitmq
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: guest
      RABBITMQ_DEFAULT_PASS: guest
    volumes:
      - rabbitdata:/var/lib/rabbitmq
    healthcheck:
      test: ["CMD", "rabbitmq-diagnostics", "check_port_connectivity"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - shopee-net

  spring-boot-app:
    image: hunglk/mini-shopee-app:latest
    container_name: shopee_mini_app
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:postgresql://postgres-db:5432/shopee_mini
      SPRING_DATASOURCE_USERNAME: sa_user
      SPRING_DATASOURCE_PASSWORD: sa_password
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_PORT: 6379
      SPRING_RABBITMQ_HOST: rabbitmq
      SPRING_RABBITMQ_PORT: 5672
      SPRING_RABBITMQ_USERNAME: guest
      SPRING_RABBITMQ_PASSWORD: guest
    depends_on:
      postgres-db:
        condition: service_healthy
      rabbitmq:
        condition: service_healthy
    networks:
      - shopee-net

networks:
  shopee-net:
    driver: bridge

volumes:
  pgdata:
  redisdata:
  rabbitdata:
Lưu ý: Cấu hình trên đã đặt sẵn image chính xác trên Docker Hub của bạn là hunglk/mini-shopee-app:latest và tự động đồng bộ hóa các tài nguyên Redis và RabbitMQ.
RabbitMQ Dashboard

4. Giám sát hàng đợi với RabbitMQ Management Dashboard

Để dễ dàng theo dõi trực quan và giám sát các hoạt động của hàng đợi bất đồng bộ (Message Queue) khi thực hiện kiểm thử đặt hàng cực hạn, RabbitMQ Management Dashboard đã được tích hợp và kích hoạt sẵn thông qua Docker trên cổng 15672.

Tính năng nổi bật của Dashboard:
  • Giám sát Queue thực tế: Xem hàng đợi shopee.order.queue hoạt động trong thời gian thực.
  • Biểu đồ lưu lượng: Hiển thị đồ thị trực quan về tốc độ đẩy tin nhắn vào (Publish) và tốc độ tiêu thụ tin nhắn của hệ thống (Deliver/Acknowledge).
  • Độ trễ thấp: Theo dõi các chỉ số về số lượng tin nhắn đang chờ xử lý (Ready) và tin nhắn đang được xử lý chưa hoàn tất (Unacked).
  • Xem nội dung tin nhắn: Cho phép Tester xem nội dung chi tiết payload JSON của các đơn hàng đang xếp hàng chờ.

5. Tham số kết nối & Cơ sở dữ liệu mẫu

Khi hệ thống khởi chạy thành công, một bộ dữ liệu kiểm thử tiêu chuẩn sẽ tự động được ghi (seed) vào cơ sở dữ liệu:

  • Tài khoản USER kiểm thử: Username user | Password password (Số dư ví: 1000.00 USD).
  • Tài khoản ADMIN kiểm thử: Username admin | Password adminpassword (Số dư ví: 0.00 USD).

6. Dừng hệ thống

Để dừng toàn bộ hệ thống và giải phóng bộ nhớ RAM cho máy tính của bạn một cách an toàn, hãy mở CMD tại thư mục làm việc và chạy lệnh: 

docker compose down

Lệnh này sẽ tắt các container một cách an toàn mà vẫn giữ nguyên dữ liệu kiểm thử đã lưu trong Database của bạn cho các lần chạy tiếp theo.
