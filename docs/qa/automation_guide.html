Automation QA

Hướng dẫn thiết lập Automation Testing

Chuyển đổi từ kiểm thử thủ công sang kiểm thử tự động là bước ngoặt quan trọng trong vòng đời phát triển phần mềm. Tài liệu này hướng dẫn từng bước xây dựng bộ automation test cho Mini Shopee — từ mức độ đơn giản nhất với Newman (Postman CLI), tiến lên RestAssured (Java), và cuối cùng là Playwright (TypeScript) để kiểm thử cả UI lẫn API. Mỗi cấp độ được thiết kế để QA có thể áp dụng ngay mà không cần kinh nghiệm automation trước đó.

🔧 Tổng quan công cụ Automation

Dưới đây là so sánh các công cụ automation phổ biến phù hợp với dự án Mini Shopee:

Công cụ Loại test Ngôn ngữ Độ khó Phù hợp với
RestAssured API Testing Java ⭐⭐ Full-Stack QA, đã biết Java cơ bản
Playwright UI + API JS / TS ⭐⭐ QA muốn dùng JS
JMeter Performance GUI / XML ⭐⭐⭐ Performance QA
Postman / Newman API JavaScript ⭐ Manual tester mới bắt đầu automation
💡 Khuyến nghị lộ trình:
Bắt đầu với Newman (Postman CLI) nếu bạn đã quen với Postman — không cần viết code, chỉ cần thêm test script vào từng request. Sau khi thành thạo, tiến lên RestAssured để có full control hơn và tích hợp tốt hơn với CI/CD pipeline.

🚀 Cấp độ 1: Automation với Newman (Postman CLI)

Newman là CLI runner chính thức của Postman. Bạn có thể chạy toàn bộ Postman Collection tự động mà không cần mở giao diện. Đây là bước khởi đầu lý tưởng nhất cho Manual Tester muốn bước vào Automation.

Bước 1 — Cài đặt Newman

Yêu cầu: Node.js >= 16 đã được cài trên máy.

npm install -g newman

Bước 2 — Chạy Collection

Export collection từ Postman (File → Export) rồi chạy lệnh sau:

newman run mini_shopee_postman_collection.json \
  --env-var "base_url=http://localhost:8080"
📝 Lưu ý: Đảm bảo ứng dụng đang chạy trên localhost:8080 trước khi thực thi lệnh trên.

Bước 3 — Thêm Test Scripts trong Postman

Với mỗi request trong Postman, mở tab Tests và viết assertion bằng JavaScript. Ví dụ assertion đơn giản nhất: 

pm.test("Status 200", () => pm.response.to.have.status(200));

Bước 4 — Xuất báo cáo HTML

Cài thêm reporter để xuất báo cáo dạng HTML đẹp mắt:

npm install -g newman-reporter-html

newman run collection.json \
  --reporters html \
  --reporter-html-export report.html

📋 Ví dụ: Test Script đầy đủ cho API Login

Dán đoạn code dưới đây vào tab Tests của request POST /api/auth/login:

// Trong tab 'Tests' của request POST /api/auth/login
pm.test("Login thành công - Status 200", function () {
    pm.response.to.have.status(200);
});

pm.test("Response có JWT token", function () {
    const jsonData = pm.response.json();
    pm.expect(jsonData.token).to.be.a('string');
    pm.expect(jsonData.token.length).to.be.above(100);
});

// Tự động lưu token vào environment variable
const token = pm.response.json().token;
pm.environment.set("jwt_token", token);
✅ Kết quả mong đợi: Sau khi chạy Newman, terminal sẽ hiển thị số lượng test passed/failed. Token JWT được lưu vào environment và các request tiếp theo có thể dùng {{jwt_token}} để xác thực.

☕ Cấp độ 2: API Test với RestAssured (Java)

RestAssured là thư viện Java mạnh mẽ để viết API test theo phong cách BDD (Given / When / Then). Phù hợp với QA đã có nền tảng Java và muốn tích hợp automation vào Maven project của Spring Boot.

Thêm Maven Dependencies

Thêm các dependency sau vào pom.xml của dự án:

<dependency>
    <groupId>io.rest-assured</groupId>
    <artifactId>rest-assured</artifactId>
    <version>5.4.0</version>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.0</version>
    <scope>test</scope>
</dependency>

📋 Ví dụ: Test Class hoàn chỉnh cho Order API

Tạo file src/test/java/com/example/OrderApiTest.java: 

@TestMethodOrder(MethodOrderer.OrderAnnotation.class)
public class OrderApiTest {

    private static String jwtToken;
    private static final String BASE_URL = "http://localhost:8080";

    @Test
    @Order(1)
    void testLogin() {
        jwtToken = given()
            .contentType("application/json")
            .body("{\"username\":\"user\",\"password\":\"password\"}")
        .when()
            .post(BASE_URL + "/api/auth/login")
        .then()
            .statusCode(200)
            .body("token", notNullValue())
            .extract().path("token");
    }

    @Test
    @Order(2)
    void testCreateOrderSuccess() {
        given()
            .header("Authorization", "Bearer " + jwtToken)
            .contentType("application/json")
            .body("{\"items\":[{\"productId\":2,\"quantity\":1}]}")
        .when()
            .post(BASE_URL + "/api/orders")
        .then()
            .statusCode(200);
    }

    @Test
    @Order(3)
    void testOrderWithInsufficientStock() {
        given()
            .header("Authorization", "Bearer " + jwtToken)
            .contentType("application/json")
            .body("{\"items\":[{\"productId\":1,\"quantity\":99999}]}")
        .when()
            .post(BASE_URL + "/api/orders")
        .then()
            .statusCode(400);
    }
}
Test Case Mô tả Expected Status Kết quả
testLogin() Đăng nhập bằng credentials hợp lệ và lấy JWT token 200 OK Token được lưu vào biến tĩnh để dùng chung
testCreateOrderSuccess() Tạo đơn hàng với sản phẩm có tồn kho đủ 200 OK Đơn hàng được tạo thành công
testOrderWithInsufficientStock() Tạo đơn hàng với số lượng vượt quá tồn kho 400 Bad Request Hệ thống từ chối và trả về lỗi

Chạy test bằng Maven

mvn test -Dtest=OrderApiTest
💡 Tip: Dùng @TestMethodOrder(MethodOrderer.OrderAnnotation.class) để đảm bảo thứ tự thực thi: login trước → lấy token → dùng token cho các test tiếp theo. Đây là pattern chuẩn cho API test có stateful session.

🎭 Cấp độ 3: UI + API Test với Playwright

Playwright hỗ trợ cả kiểm thử UI (browser automation) lẫn API trong cùng một framework. Với TypeScript, bạn có type-safety và intellisense đầy đủ khi viết test.

Khởi tạo dự án Playwright

npm init playwright@latest

Lệnh này sẽ tạo cấu trúc thư mục chuẩn gồm tests/, playwright.config.ts, và cài sẵn các browser (Chromium, Firefox, WebKit).

📋 Ví dụ: Kiểm thử luồng mua hàng đầy đủ (API mode)

Tạo file tests/shopping-flow.spec.ts:

import { test, expect } from '@playwright/test';

test('Luồng mua hàng đầy đủ', async ({ request }) => {
  // Step 1: Login
  const loginRes = await request.post('http://localhost:8080/api/auth/login', {
    data: { username: 'user', password: 'password' }
  });
  expect(loginRes.ok()).toBeTruthy();
  const { token } = await loginRes.json();

  // Step 2: Get products
  const productsRes = await request.get('http://localhost:8080/api/products', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  expect(productsRes.status()).toBe(200);

  // Step 3: Place order
  const orderRes = await request.post('http://localhost:8080/api/orders', {
    headers: { 'Authorization': `Bearer ${token}` },
    data: { items: [{ productId: 2, quantity: 1 }] }
  });
  expect(orderRes.status()).toBe(200);
});

Chạy test Playwright

# Chạy tất cả test
npx playwright test

# Chạy với UI mode (có giao diện trực quan)
npx playwright test --ui

# Xuất báo cáo HTML
npx playwright show-report
🌟 Điểm mạnh của Playwright: Một file test có thể cover cả API (dùng request fixture) lẫn UI (dùng page fixture). Bạn có thể viết test E2E kiểm tra từ giao diện người dùng cho đến phản hồi API trong một kịch bản thống nhất.

🗺️ Lộ trình tiến hóa Automation

Kế hoạch 3 tháng được đề xuất để xây dựng bộ automation test từ đầu cho dự án Mini Shopee:

Tuần 1 – 2 Newman

Viết Postman test scripts & chạy bằng Newman

  • Thêm test script vào tất cả request trong Postman Collection
  • Phủ toàn bộ happy path: GET /products, POST /orders, GET /orders/{id}, ...
  • Cài Newman và chạy collection từ terminal thành công
  • Tạo báo cáo HTML và lưu vào thư mục reports/
newman run collection.json \
  --env-var "base_url=http://localhost:8080" \
  --reporters html \
  --reporter-html-export reports/week1.html
Tuần 3 – 4 RestAssured

Setup RestAssured & viết 20 test cases cơ bản

  • Tạo Maven module riêng cho automation test (api-tests/)
  • Viết test cho: Login, Register, Product list, Product detail, Create order, Get orders
  • Đạt mốc 20 test cases pass ổn định
  • Thiết lập @BeforeAll để login một lần, dùng token cho tất cả test
Tháng 2 Negative & Concurrency

Viết test cho negative cases và concurrency scenarios

  • Negative cases: đặt hàng quá tồn kho, đăng nhập sai mật khẩu, token hết hạn, ...
  • Concurrency test: gửi 10 request đặt hàng đồng thời, kiểm tra chỉ 1 request thành công
  • Dùng ExecutorService hoặc CompletableFuture trong Java để simulate concurrent requests
  • Mục tiêu: phát hiện race condition và lỗi business logic dưới tải
// Ví dụ: Concurrent order test
ExecutorService pool = Executors.newFixedThreadPool(10);
List<Future<Integer>> futures = new ArrayList<>();
for (int i = 0; i < 10; i++) {
    futures.add(pool.submit(() -> placeOrder(jwtToken)));
}
long successCount = futures.stream()
    .map(f -> f.get())
    .filter(status -> status == 200)
    .count();
assertEquals(1, successCount, "Chỉ 1 request nên thành công!");
Tháng 3 CI/CD

Tích hợp CI/CD với GitHub Actions

  • Tạo workflow .github/workflows/api-test.yml
  • Cấu hình: mỗi lần push hoặc pull_request lên main → tự động chạy toàn bộ test suite
  • Hiển thị badge test status trên README
  • Gửi thông báo Slack/Email khi test fail
# .github/workflows/api-test.yml
name: API Automation Test
on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
      - name: Start application
        run: mvn spring-boot:run &
      - name: Wait for app to start
        run: sleep 30
      - name: Run API tests
        run: mvn test -Dtest=OrderApiTest,ProductApiTest
🎯 Mục tiêu cuối lộ trình: Sau 3 tháng, mỗi lần developer push code lên GitHub, toàn bộ test suite sẽ chạy tự động trong vòng dưới 5 phút và thông báo ngay nếu có regression. Đây là nền tảng vững chắc để tiến lên continuous delivery.
