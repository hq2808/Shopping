Test Case Specification

Ma Trận Danh Sách Test Cases
(Structured Test Case Matrix)

Bảng tổng hợp chi tiết tất cả các ca kiểm thử của dự án được phân loại theo cấu phần chức năng, mô tả đầy đủ các bước thực hiện và kết quả mong đợi trên cả API và DB.

Phần 1

1. Nhóm Xác Thực & Tài Khoản (Authentication & Profile)

TC ID Mô tả kịch bản Các bước thực hiện (Steps) Kết quả API mong đợi Kiểm tra Database (PostgreSQL)
TC-AUTH-01 Đăng ký tài khoản hợp lệ Gọi POST /api/auth/register với dữ liệu username, email, password hợp lệ. HTTP 201 Created. Trả về thông tin User. SELECT * FROM users WHERE username = '...';
Trạng thái: 1 dòng mới, mật khẩu phải được băm BCrypt.
TC-AUTH-02 Đăng ký trùng tên đăng nhập Gửi lại request đăng ký với username đã tồn tại trong DB. HTTP 400 Bad Request.
Message: "Username is already taken".		 SELECT count(*) FROM users;
Trạng thái: Số lượng dòng không đổi, không ghi nhận trùng lặp.
TC-AUTH-03 Đăng ký thiếu trường bắt buộc Gửi request đăng ký với `username` hoặc `password` để trống. HTTP 400 Bad Request. Báo lỗi validation tương ứng. Không ghi nhận thêm dòng mới trong DB.
TC-AUTH-04 Đăng ký với mật khẩu ngắn Gửi password có độ dài nhỏ hơn 6 ký tự. HTTP 400 Bad Request.
Message: "Password must be at least 6 characters".		 Không ghi nhận thêm dòng mới trong DB.
TC-AUTH-05 Đăng nhập thành công Gọi POST /api/auth/login với username và password chính xác. HTTP 200 OK. Trả về chuỗi Token JWT hợp lệ. -
TC-AUTH-06 Đăng nhập sai mật khẩu Gọi login với đúng username nhưng sai password. HTTP 401 Unauthorized.
Message: "Invalid username or password".		 -
TC-PROFILE-01 Xem Profile có JWT hợp lệ Gọi GET /api/users/profile kèm Header Authorization Bearer. HTTP 200 OK. Trả về đúng thông tin username, email, balance. Đối soát số tiền ví khớp 100% với cột `balance` của user trong DB.
Phần 2

2. Nhóm Quản Lý Sản Phẩm (Product Management CRUD)

TC ID Mô tả kịch bản Các bước thực hiện (Steps) Kết quả API mong đợi Kiểm tra CSDL & Cache
TC-PROD-01 Xem danh sách sản phẩm Gọi GET /api/products (Public API). HTTP 200 OK. Trả về danh sách sản phẩm phân trang. Dữ liệu hiển thị phải khớp các sản phẩm hiện có trong DB.
TC-PROD-02 Xem chi tiết sản phẩm hợp lệ Gọi GET /api/products/{id} với ID có sẵn. HTTP 200 OK. Trả về thông tin chi tiết sản phẩm. -
TC-PROD-03 Xem sản phẩm không tồn tại Gọi GET /api/products/999. HTTP 404 Not Found.
Message: "Product not found...".		 -
TC-PROD-04 Admin tạo sản phẩm mới Gọi POST /api/products bằng token ADMIN với body hợp lệ. HTTP 201 Created. Trả về thông tin sản phẩm có ID mới. 1. DB PostgreSQL có thêm dòng mới.
2. Khóa product:stock:{id} trên Redis được tạo và lưu đúng số kho.
TC-PROD-05 Admin tạo sản phẩm lỗi Tạo sản phẩm với `price = -10` hoặc `stock = -5`. HTTP 400 Bad Request. Báo lỗi validation đầu vào. Không có bản ghi nào được lưu trong DB hay Redis.
TC-PROD-06 Admin cập nhật sản phẩm Gọi PUT /api/products/{id} bằng token ADMIN. HTTP 200 OK. Trả về sản phẩm đã cập nhật. DB và Redis RAM được cập nhật đồng nhất thông tin mới.
TC-PROD-07 Admin xóa sản phẩm chưa mua Gọi DELETE /api/products/{id} đối với sản phẩm chưa từng được đặt. HTTP 204 No Content. 1. DB không còn dòng sản phẩm này.
2. Khóa sản phẩm trên Redis RAM bị thu hồi (evicted).
TC-PROD-08 Admin xóa sản phẩm đã được mua Gọi DELETE /api/products/1 đối với sản phẩm đã phát sinh đơn hàng. HTTP 500 Internal Server Error (lỗi khóa ngoại). Sản phẩm vẫn nguyên vẹn trong DB và Redis để bảo toàn lịch sử hóa đơn.
Phần 3

3. Nhóm Quy Trình Đặt Hàng & Giao Dịch (Order Processing)

TC ID Mô tả kịch bản Các bước thực hiện (Steps) Kết quả API mong đợi Kiểm tra CSDL & Cache
TC-ORDER-01 Đặt hàng đồng bộ thành công Dùng token USER gọi POST /api/orders với mã sản phẩm và tồn kho hợp lệ. HTTP 201 Created. Trả về thông tin đơn hàng có trạng thái PAID. 1. Số dư ví người dùng trong DB giảm.
2. Tồn kho sản phẩm trong DB giảm.
3. Sinh hóa đơn trong `orders` và `order_items`.
TC-ORDER-02 Đặt hàng đồng bộ vượt quá số dư ví Đặt đơn hàng có tổng trị giá lớn hơn số dư ví hiện tại của User. HTTP 400 Bad Request. Báo không đủ số dư ví. DB giữ nguyên số tiền ví của User và tồn kho của sản phẩm (không trừ tiền/kho). Không sinh đơn hàng.
TC-ORDER-03 Đặt hàng đồng bộ vượt quá tồn kho Đặt đơn hàng mua số lượng sản phẩm lớn hơn số lượng tồn kho thực tế. HTTP 400 Bad Request. Báo hết hàng. Kho và ví giữ nguyên. Không sinh đơn hàng.
TC-ORDER-04 Đơn hàng tự động Rollback khi có lỗi Đặt đơn hàng gồm 2 sản phẩm: AirPods Pro (đủ kho) và iPad Air (vượt quá kho). HTTP 400 Bad Request. Báo iPad Air hết hàng. Kiểm thử tính nguyên tử: Tồn kho của AirPods Pro và ví tiền người dùng phải giữ nguyên vẹn (rollback thành công).
TC-ORDER-05 Đặt đơn hàng rỗng hoặc quá lớn Gửi đơn hàng trống hoặc chứa hơn 50 dòng sản phẩm. HTTP 400 Bad Request. DB giữ nguyên trạng.
TC-ORDER-06 Đặt đơn bất đồng bộ thành công Dùng token USER gọi POST /api/orders/async mua AirPods Pro hợp lệ. HTTP 202 Accepted. Trả về hóa đơn có status PENDING và orderId = null. 1. Redis (Tức thì): Ví người dùng và kho sản phẩm bị trừ ngay trên RAM.
2. DB (Sau 1-2s): Hàng đợi RabbitMQ đẩy đơn ghi nhận xuống PostgreSQL DB, status đơn chuyển sang PAID.
TC-ORDER-07 Đặt đơn bất đồng bộ thất bại Gửi request bất đồng bộ vượt quá số dư hoặc tồn kho hiện tại trên Redis. HTTP 400 Bad Request (Bị chặn ngay tại tầng Redis Lua Script). Redis và DB giữ nguyên trạng thái, không khấu trừ và không đẩy tin nhắn vào RabbitMQ.
Phần 4

4. Nhóm Phân Quyền Vai Trò (Role-Based Access Control)

TC ID Mô tả kịch bản Các bước thực hiện (Steps) Kết quả API mong đợi Xác minh an ninh hệ thống
TC-RBAC-01 Truy cập không cung cấp Token Gọi API được bảo vệ (ví dụ lấy danh sách orders) không kèm header Authorization. HTTP 403 Forbidden hoặc 401 Unauthorized. Hệ thống Spring Security chặn đứng từ tầng lọc, không cho phép truy cập tài nguyên.
TC-RBAC-02 USER gọi API của ADMIN Dùng token ROLE_USER gọi các lệnh POST/PUT/DELETE sản phẩm. HTTP 403 Forbidden.
Message: "Access denied...".		 DB không thay đổi dữ liệu sản phẩm, yêu cầu bị chặn đứng hoàn toàn.
TC-RBAC-03 ADMIN gọi API đặt hàng của USER Dùng token ROLE_ADMIN gọi API đặt hàng POST /api/orders. HTTP 403 Forbidden. Admin không được phép đặt hàng, DB không sinh đơn hàng.
